To begin developing with AWS via CLI (Command Line Interpreter):

{% highlight bash %} $ sudo easy_install pip {% endhighlight %}

Some requirements during installation...

{% highlight bash %} $ sudo easy_install nose $ sudo easy_install tornado {% endhighlight %}

Rerun installation, ignoring already installed packages.

{% highlight bash %} $ sudo -H pip install awscli --upgrade --ignore-installed six {% endhighlight %}

Check if we successfully install

{% highlight bash %} $ aws --version aws-cli/1.15.28 Python/2.7.10 Darwin/17.4.0 botocore/1.10.28 {% endhighlight %}

Sign in to your AWS console, and register an access key via Security Credentials. Since my region is located in Tokyo, I chose ap-northeast-1. And response format as json.

{% highlight bash %} $ aws configure AWSAccessKeyId= AWSSecretKey= Region=ap-northeast-1 Format=json {% endhighlight %}

Test our configuration, we'll receive an empty json formatted response, as we don't have any ec2 instances running yet.

{% highlight bash %} $ aws ec2 describe-instances { "Reservations": [] } {% endhighlight %}
